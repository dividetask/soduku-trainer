# Roadmap

Version 1 is deliberately small: get the core play loop right before
adding intelligence. This document captures what is in v1 and what is
planned for later.

## Version 1 — Core Play Loop

Included:

- **Main Menu** with three difficulty buttons: Easy, Medium, Hard.
  - All three start a puzzle drawn from the same pre-generated pool in
    v1. The buttons exist as stable UI entry points so future versions
    can tune difficulty without moving controls around.
- **Pre-generated puzzle set** (150 puzzles, generated by us via an
  offline developer tool) bundled with the app. No on-device generator
  in v1 — see v3. Details in [`tech-stack.md`](tech-stack.md).
- **Game Screen** with:
  - 9x9 board with givens in black.
  - Toggle Pencil.
  - Color picker (8 colors, default blue, reset to blue on every new
    game; black reserved for givens).
  - Digit buttons 1–9 with per-digit highlighting. Exhausted digits
    (nine instances placed) render in grey but remain selectable.
  - Single-tap to place, double-tap to clear. Single-tap never
    overwrites an existing value. Pencil-mode taps on filled cells are
    ignored.
  - Undo with full history back to the starting position.
  - Give Up, which ends the game and returns to the Main Menu without
    revealing the solution.
  - Solve, which replaces the Give Up label once the puzzle is mostly
    solved (≥70/81 correct cells **and** no incorrect entries). Fills
    remaining cells in a distinct "solve" color, waits 5 seconds, then
    returns to the Main Menu.
  - Hint button present in the UI but **non-functional** in v1.

Explicitly **not** in v1:

- Functional hints / teaching of solving techniques.
- Difficulty tuning per button.
- Timers, scoring, streaks, leaderboards.
- Save / resume across sessions, account sync, cloud state.
- Multiplayer or shared puzzles.
- Custom color palettes or themes.
- Dark mode.
- On-device puzzle generation.

Permanent non-goals (all versions):

- **Conflict warnings / error highlighting.** The game never flags
  incorrect entries. Discovering and correcting mistakes is part of
  the training.

## Version 2 — Hints

Turn the Hint button into a working feature. Target capabilities:

- A solver that can identify which technique applies next (naked
  single, hidden single, naked/hidden pair, pointing pair, box/line
  reduction, X-wing, etc.).
- Progressive disclosure: first hint names the technique, second hint
  highlights the relevant row/column/box, third hint points at the
  target cell(s), fourth hint fills it in.
- No-op (or a friendly "no hint available") when the board state is
  inconsistent with the puzzle's solution.

Candidates to consider alongside v2 (not committed):

- **Dark mode** — Add a theme toggle. Requires tuning the 8 guess
  colors and givens rendering for both themes.
- **"Mostly solved" threshold tuning** — Possibly raise, lower, or
  vary the 70/81 threshold per difficulty.

## Version 3 — Difficulty Tuning & On-device Generator

- Introduce an **on-device puzzle generator** so puzzles no longer
  come solely from the bundled pool.
- Give each difficulty button a distinct puzzle profile:
  - **Easy** — Solvable with only naked/hidden singles.
  - **Medium** — Requires intermediate techniques (pairs, pointing).
  - **Hard** — Requires advanced techniques (X-wing, swordfish,
    coloring).

Difficulty tuning depends on the v2 solver, because difficulty is
defined by which techniques are required to solve a given puzzle.

## Later / Speculative

- **Trainer mode** — Present puzzles that specifically require one
  technique, to let the player drill that technique.
- **Statistics** — Track which techniques the player uses (via Hint)
  and which they tend to miss.
- **Save / resume** — Persist the current game across sessions.
- **Timer & scoring** — Optional, off by default.
- **Themes / palettes** — Customizable color picker.
